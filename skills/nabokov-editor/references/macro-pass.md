# Macro pass — essays and arguments

A structural read for essays, opinion pieces, and academic texts (not READMEs or
reference docs). Sources: Williams (*Style*), Zinsser, the Harvard/Purdue
writing guides. Run it after the judgment pass, before fixing.

- **Thesis**: is the central claim *arguable* (answers how/why), or merely
  descriptive (answers who/what/when)? A descriptive thesis turns the essay into
  a report — flag it and propose a sharper claim for the user to approve.
- **Reverse outline**: write down each paragraph's one main point. The list
  should form a chain with no repeats and no dropped threads; a paragraph with
  two points wants splitting, two paragraphs with one point want merging
  (approval gate).
- **Stitching over signposting**: paragraph openers like *Moreover /
  Furthermore / Additionally* are mechanical connectors. The fix is not deleting
  the word — it's opening with a semantic echo of the previous paragraph's key
  concept, so the logic carries the transition.
- **Old-to-new flow** (cohesion): sentences should open with information the
  reader already has and end with the new. This is also the legitimate use of
  passive voice — keep a passive that fronts the known topic.
- **Conclusion**: synthesize, don't summarize. No new evidence, no apologies
  ("this is just one approach…"), no hollow closers. If the conclusion restates
  the intro, propose one of: zoom out to the bigger picture, name a consequence
  for the future, or end on the strongest concrete image.
